================================================================================
LOCKED DECISIONS (reference during build)
================================================================================

SET LOGGING (Phase 1 baseline; Phase 2.5 extends — see PHASE 2.5)
- Each set tagged: working | warmup (default: working)
  - Phase 2.5b adds technique tags on working rows: none | failure | drop | …
- Only non-warmup sets count toward volume; warmups excluded from analytics
- Effort on working sets (Phase 2.5):
  - Encouraged during logging; NOT required before set-complete ✓
  - Save warns if working sets missing effort; user may save anyway
  - Warmups: effort always optional
- Effort metric (Phase 2.5b): global user setting rpe | rir
  - UI: chip select (no typing) — RPE 6–10 (0.5 steps) or RIR 4+, 3, 2, 1, 0
  - Store canonical sets.rpe (6–10); RIR→RPE on write: 4+→6, 3→7, 2→8, 1→9, 0→10
  - Optional sets.effort_rir for display fidelity
- e1RM best-set selection (Phase 2.5b): technique none | failure | amrap | myo_activation
  - Exclude drop segments and myo_mini from e1RM
- Unilateral (Phase 2.5b): exercises.logging_mode + sets.side when per_side

TIME BUCKETS
- Hybrid model:
  - Calendar week → weekly_volume, per-lift trends, stall detection, performance drop
  - Rolling 7-day → fatigue_signals, RPE trend, deload score
- User setting: first day of week (default Monday)
- UI labels must differ: "Week of Jun 2" vs "Last 7 days"

WEIGHT
- Store canonical kg in database (numeric)
- User setting: display unit kg | lbs (convert on read/write in UI)

e1RM & PERFORMANCE DROP
- Input: best working set per exercise per session, reps ≤ 10 only
- Formula selected by exercise.movement_type:
  - upper_push    → Mayhew:   w / (0.522 + 0.419 × e^(-0.055 × r))
  - upper_pull    → Epley:    w × (1 + r/30)
  - lower_compound → Lombardi: w × r^0.10
  - lower_hinge   → Brzycki:  w / (1.0278 - 0.0278 × r)
  - isolation     → Wathen:   w / (0.488 + 0.538 × e^(-0.075 × r))
  - default       → Epley (fallback if movement_type unset)
- performance_drop_detected: e1RM down ≥3% for 2 consecutive calendar weeks
- Applies to exercises user marks as tracked for progression (anchor flag)

ANCHOR / TRACKED EXERCISES
- User explicitly marks which exercises drive progression charts, drop detection,
  and weekly summary callouts (can mark all lifts if desired)
- Not auto-limited to compounds

MUSCLE ATTRIBUTION (volume analytics)
- exercises: primary_muscle (required) + secondary_muscles[] (max 2) + other_muscles[] (max 3)
- Weights: primary 1.0, secondary 0.5, other 0.3 on weighted tonnage only
- muscle_groups[] synced via DB trigger for search/filter compat
- Headline weekly volume = raw total (no double-count); per-muscle heatmap = weighted

DELOAD RECOMMENDATION
- Tiered score 0–100 → lean | moderate | strong deload
- Inputs (weighted): rolling 7-day avg RPE, RPE trend, performance_drop,
  consecutive high-RPE sessions (RPE ≥ 8.5 on working sets)
- Plain-language reason always shown alongside score
- NOT a hard AND gate (original spec superseded)

TRAINING BLOCKS
- MVP: manual start/end dates + optional name/notes
- Future: auto-suggest boundaries from deload events

SESSION & PROGRAMS
- Freeform workout logging (add exercises ad hoc each session)
- Stored workout programs:
  - Phase 1: flat ordered exercise list, no prescribed sets
  - Phase 2.5a: program_days — one program, multiple named days (e.g. PPL), exercises per day
  - Phase 3+: full prescription (sets/reps/RPE targets) + multi-week mesocycles

ANALYTICS UI
- Chart time range: user setting (default 6 weeks on first launch)

CLIENT STATE
- Server state: TanStack Query
- Active workout session: Zustand

AUTH (recommended — user asked)
- Supabase Auth: email + password
- Google OAuth optional later; not MVP

INFRASTRUCTURE / DATA STORAGE
- Database: Supabase Postgres (all workout, exercise, set, analytics tables)
- Auth: Supabase Auth (user_id on all rows; RLS per user)
- Realtime: Supabase Realtime (optional — nice for sync, not required MVP)
- No custom backend server for MVP
- Analytics refresh: Postgres function or Supabase Edge Function triggered on
  workout save (recalculate weekly_volume, exercise_performance, fatigue_signals)
- File storage: none for MVP (no videos/images). Supabase Storage reserved for
  future export/import or attachments.

TECH STACK (unchanged from spec)
- Frontend: React + Vite (or Next.js — pick at Phase 0) + shadcn/ui + Tailwind
- Charts: Recharts
- Backend: Supabase only

OUT OF SCOPE (all phases until explicitly promoted)
- Social, streaks, badges
- Video exercise library
- Nutrition tracking
- iOS native app
- Multi-user / trainer mode
- Apple Health / HRV (recovery_metrics table stubbed for future)

LOCKED DECISIONS (grill session — Jun 2026)

Focus logger (2.5a):
- Mobile-only focus mode default (<sm / touch-primary): one exercise at a time,
  swipe or progress dots to jump; completed exercises collapse to compact strip
- Desktop keeps multi-exercise scroll list (no forced focus mode)
- Session metadata (date, duration, notes) collapsible during active logging
- Autofill: full last-session pattern — all set rows (count, type, weight, reps,
  effort) as starting placeholders when adding exercise or starting program day
- Rest timer: auto-start on set ✓ when users_settings.rest_timer_enabled
  - Cascade: exercises.default_rest_seconds → users_settings.rest_timer_seconds
    → 90s default
  - Drop-set skip (no rest between consecutive drops) ships with 2.5b set types

Programs (2.5a):
- program_days table: program_id, name, sort_order
- program_exercises references program_day_id (not program_id directly)
- Start workout: pick program → pick day → preload that day's exercises

Effort (2.5a behavior change + 2.5b RIR):
- Optional during session; confirmation on save if working sets lack effort
- Global effort_metric in users_settings: rpe | rir (default rpe)

Set types (2.5b v1):
- Keep set_type (or rename set_kind): warmup | working for analytics gate
- sets.technique on working rows: none | failure | drop (enum; extend in 2.5b.2)
- UI: tap set # → bottom sheet; keep W/Work quick pills for warmup/working
- Failure: optional auto-suggest effort RIR 0 / RPE 10 (implement in 2.5b)

Analytics (2.5b):
- Volume: all non-warmup rows at logged weight × reps (full tonnage)
- e1RM / best-set: technique in (none, failure, amrap, myo_activation); exclude drop
- Fatigue / deload: avg RPE on working sets where effort logged (nulls skipped)
- Unilateral alternating mode: volume × 2 for parity with bilateral barbell work

Unilateral (2.5b):
- exercises.logging_mode: bilateral | unilateral_alternating | unilateral_per_side
- sets.side: left | right | both | null when unilateral_per_side
- Weight field = one dumbbell load for per_side; reps = that side

Myo-reps (2.5b.2 — fast follow after core set types):
- Grouped block UI; flat rows linked by sets.set_group_id + set_group_seq
- Roles via technique: myo_activation | myo_mini
- Short rest (~15s) between minis; long rest after group completes
- e1RM from activation only; all segments count toward volume

Out of scope for 2.5:
- AI mesocycles, set/rep/RPE prescription targets (Phase 3)
- Social, video library, rest-pause / AMRAP / backoff set types (defer post-2.5b.2)
- Hard-set discount factors (Reps-app 0.5× drops) — defer until deload tuning
