# Workout Tracking & Analytics App

================================================================================
LOCKED DECISIONS (reference during build)
================================================================================

SET LOGGING
- Each set tagged: working | warmup (default: working)
- Only working sets count toward volume, RPE averages, e1RM, fatigue
- RPE required on working sets; optional on warmups
- RPE precision: 0.5 increments (6, 6.5, 7 … 10)

TIME BUCKETS
- Hybrid model:
  - Calendar week → weekly_volume, per-lift trends, stall detection, performance drop
  - Rolling 7-day → fatigue_signals, RPE trend, deload score
- User setting: first day of week (default Monday)
- UI labels must differ: "Week of Jun 2" vs "Last 7 days"

WEIGHT
- Store canonical kg in database (numeric)
- User setting: display unit kg | lbs (convert on read/write in UI)

e1RM & PERFORMANCE DROP
- Input: best working set per exercise per session, reps ≤ 10 only
- Formula selected by exercise.movement_type:
  - upper_push    → Mayhew:   w / (0.522 + 0.419 × e^(-0.055 × r))
  - upper_pull    → Epley:    w × (1 + r/30)
  - lower_compound → Lombardi: w × r^0.10
  - lower_hinge   → Brzycki:  w / (1.0278 - 0.0278 × r)
  - isolation     → Wathen:   w / (0.488 + 0.538 × e^(-0.075 × r))
  - default       → Epley (fallback if movement_type unset)
- performance_drop_detected: e1RM down ≥3% for 2 consecutive calendar weeks
- Applies to exercises user marks as tracked for progression (anchor flag)

ANCHOR / TRACKED EXERCISES
- User explicitly marks which exercises drive progression charts, drop detection,
  and weekly summary callouts (can mark all lifts if desired)
- Not auto-limited to compounds

DELOAD RECOMMENDATION
- Tiered score 0–100 → lean | moderate | strong deload
- Inputs (weighted): rolling 7-day avg RPE, RPE trend, performance_drop,
  consecutive high-RPE sessions (RPE ≥ 8.5 on working sets)
- Plain-language reason always shown alongside score
- NOT a hard AND gate (original spec superseded)

TRAINING BLOCKS
- MVP: manual start/end dates + optional name/notes
- Future: auto-suggest boundaries from deload events

SESSION & PROGRAMS
- Freeform workout logging (add exercises ad hoc each session)
- Stored workout programs (MVP): named ordered exercise list, no prescribed sets
- Phase 3+: full prescription (sets/reps/RPE targets) + multi-week mesocycles

ANALYTICS UI
- Chart time range: user setting (default 6 weeks on first launch)

CLIENT STATE
- Server state: TanStack Query
- Active workout session: Zustand

AUTH (recommended — user asked)
- Supabase Auth: email + password
- Google OAuth optional later; not MVP

INFRASTRUCTURE / DATA STORAGE
- Database: Supabase Postgres (all workout, exercise, set, analytics tables)
- Auth: Supabase Auth (user_id on all rows; RLS per user)
- Realtime: Supabase Realtime (optional — nice for sync, not required MVP)
- No custom backend server for MVP
- Analytics refresh: Postgres function or Supabase Edge Function triggered on
  workout save (recalculate weekly_volume, exercise_performance, fatigue_signals)
- File storage: none for MVP (no videos/images). Supabase Storage reserved for
  future export/import or attachments.

TECH STACK (unchanged from spec)
- Frontend: React + Vite (or Next.js — pick at Phase 0) + shadcn/ui + Tailwind
- Charts: Recharts
- Backend: Supabase only

OUT OF SCOPE (all phases until explicitly promoted)
- Social, streaks, badges
- Video exercise library
- Nutrition tracking
- iOS native app
- Multi-user / trainer mode
- Apple Health / HRV (recovery_metrics table stubbed for future)

PHASE 2 — ANALYTICS DASHBOARD
================================================================================
Goal: Answer "Am I progressing?" and "Should I deload?" in plain language.

Depends on: Phase 1 (≥3 weeks of real data recommended for meaningful charts)

Tasks:
[ ] Analytics refresh (complete implementation):
    - weekly_volume on calendar week boundary (respect user week_start_day)
    - exercise_performance per exercise per week (e1RM, best set, volume)
    - fatigue_signals on rolling 7-day window
    - deload_score + deload_tier + reason string
[ ] Dashboard layout (data-forward, shadcn, no fluff)

Charts (Recharts, range from user setting — default 6 weeks):
[ ] Weekly volume: tonnage + total sets (toggle), compound vs total
[ ] Per-lift progress: e1RM over time for is_tracked exercises
[ ] RPE trend: rolling 7-day avg with calendar week overlay optional

Cards:
[ ] Fatigue / deload card: score, tier (lean/moderate/strong), plain-language reason
[ ] Weekly summary (template-based NLG MVP, no LLM required):
    - Tracked lifts: up / flat / down vs prior week
    - Volume trend by muscle group
    - RPE direction (last 7 days)
    - Example: "Squat e1RM up 2%. Chest volume flat. RPE trending high — lean deload."

Progression logic:
[ ] Stall flag: same weight × reps 2+ weeks at similar RPE (±0.5)
[ ] performance_drop_detected per tracked exercise (e1RM ≥3% down 2 weeks)

Training blocks:
[ ] Manual block create/end (name, dates, notes)
[ ] Block summary: total volume, sets, avg RPE, PRs, top exercises
[ ] Compare any two blocks: volume % delta, RPE delta, PRs

Exit criteria:
- After logging, dashboard updates within one refresh cycle
- Deload card produces sensible tier + explanation on test scenarios
- Weekly summary readable without interpreting raw charts

PHASE 3 — AI PROGRAMMING
================================================================================
Goal: Mesocycle planning + auto-regulation based on your own history.

Depends on: Phase 2 trustworthy for ≥1 full mesocycle of personal use

Tasks:
[ ] Extend programs to full prescription:
    - Per exercise: target sets, reps range, RPE target per week
[ ] Mesocycle entity (4–6 weeks):
    - Week-by-week volume targets
    - Progressive overload rules
[ ] AI plan generation:
    - Input: training history, tracked lift trends, goals, days/week, equipment
    - Output: mesocycle plan with progressive volume
[ ] Auto-regulate:
    - After each week: adjust next week from RPE + performance signals
[ ] Block debrief:
    - LLM summary: what worked, what stalled, suggested next block focus
[ ] Program evolution: MVP exercise-list programs upgrade to full prescriptions

Exit criteria:
- Generate a 4-week plan from real history
- Week 2 plan adjusts based on week 1 RPE/performance
- Block debrief summarizes completed manual or AI block
